> 一个基于**多智能体协作**的算法竞赛辅导 Web 系统。学生提交一道算法题，
> 一支「AI 教练团队」协同完成：题目分析 → 策略规划 → 苏格拉底式引导 → 代码审查 → 自动判题，

## 一、系统概述

ARENA 把「算法竞赛辅导」这件事，拆解给一支各司其职的 AI 智能体团队来协作完成。
与「一个大模型对话框」不同，本系统的核心是**多智能体编排 + 真实代码判题 + 个性化教学**：

- 不是让模型直接给答案，而是用**苏格拉底教学法**分层引导学生自己想出来；
- 不是「看起来能跑」，而是把代码丢进**真实沙箱**跑测试用例判 AC/WA/TLE/RE/CE；
- 不是千人一面，而是根据学生的**历史画像**自适应调整教学策略（因材施教）。

整个流程对学生**全程可视化**：右侧的智能体团队会随工作进度逐个点亮，配合流式打字机输出，
让「多智能体协作」这件事看得见、摸得着。

## 二、功能模块详解

### 2.1 多智能体协作团队

系统由 **7 个协作角色**组成（RAG 检索 + 分析/规划/导师/审查 LLM 智能体 + 判题官 + ReAct 调试 agent）：

| 智能体 | 职责 | 产出 |
|--------|------|------|
| 🔍 题库检索官 | RAG 相似题检索 | 从题库召回最相关的历史题，举一反三 |
| 🧠 题目分析师 | 拆解题目本质 | 题型、难度、目标复杂度、核心突破口、易错点、知识点 |
| 🗺️ 策略规划师 | 解法谱系 | 多种解法 + 时空复杂度 + 推荐指数 + 适用场景 + 学习路径 |
| 🎓 苏格拉底导师 | 分层引导 / 答疑 | 4 层递进式提示；代码感知的自由对话 |
| 🔬 代码审查师 | 静态审查 | verdict、bug 定位、复杂度判断、优化建议、综合评分 |
| ⚖️ 判题官 | 双轨判题 | 有官方数据走**真值判定**，无数据自动**差分对拍**，给出最小反例 |
| 🛠️ 调试工程师 | ReAct 实验定位 bug | 拿到反例后多轮调用沙箱做实验，逐步缩小到根因 + 修复方向 |

> 判题官与调试工程师是本轮**核心升级**：把判题从"让 LLM 想象期望输出"换成**有真值的判定**，
> 并让调试 agent **真正动手运行实验**而非空想。详见 [§2.4 双轨判题](#24-双轨判题真值--对拍) 与
> [§2.11 Agentic 调试回路](#211-agentic-调试回路react)，以及独立的《设计说明 DESIGN.md》。

**协作可视化**：右栏「智能体团队」平时是一条迷你状态条（7 个小图标），
分析/评测/调试时对应图标**自动点亮 + 脉冲**，并自动展开成完整列表；结束后自动收起。
用户也可点击钉住展开，状态记忆到本地。

### 2.2 LangGraph 工作流编排

用状态机（StateGraph）编排两条多步骤工作流，含**条件分支**：

**① 分析流**
```
检索 → 分析 →（是有效算法题？）
                ├─ 是 → 策略规划 → 结束
                └─ 否 → 直接结束（给友好提示，引导去导师对话）
```

**② 评测流**
```
代码审查 →（语法是否通过？）
            ├─ 否 → 直接汇总（跳过判题）
            └─ 是 → 双轨判题（真值 / 对拍）→ 汇总评分
```

每条工作流用 `graph.stream(stream_mode="updates")` **逐节点产出中间结果**，
驱动前端的智能体点亮与流式渲染。

### 2.3 RAG 题库（检索增强）

- 基于 **LangChain 的 TFIDFRetriever** 建立题库索引，底层 scikit-learn，**零模型下载、开箱即用**；
- 中文采用字符级 n-gram（`char_wb`）做特征，规避中文分词依赖；
- 题库共 **114 道**：14 道手写经典题 + **100 道从公开数据集（DeepMind CodeContests）导入的真实竞赛题**
  （Codeforces 等，含原始题面、官方标签、难度分），且**每道真题都带官方测试数据**——这正是真值判题的来源；
- 导入走 `scripts/ingest_dataset.py`（HuggingFace 国内镜像流式拉取，题面进 `problems.json`、测试数据落 `data/tests/`）；
- 分析一道新题时自动召回最相似的历史题，支持「举一反三」式跳转；
- **自建题自动入库**：用户粘贴的新题被判定为有效后，会用分析师起的标题自动命名、纳入「我的题目」分组，下次可直接选用；
- **题库抽屉 + AC 标记**：题库从**左侧滑出为抽屉式面板**（背景压暗聚焦），按题型分组、可搜索筛选、带掌握进度条；用户通过（全部用例 AC）的题目标 ✓，一眼看到自己攻克了哪些。

### 2.4 双轨判题（真值 / 对拍）

判题不再依赖「让 LLM 想象期望输出」（那样 AC 的含义只是"模型觉得你对"），而是建立**真正的真值来源**，
按题目是否带官方数据自动选轨：

**轨道一 · 真值判定（truth）** —— 题目带官方测试数据时
- 直接拿**真实测试数据**逐组比对，确定性最硬，等同线上 OJ；
- 100 道导入真题全部走这一轨。

**轨道二 · 差分对拍（stress）** —— 用户自己粘的题、无官方数据时
- 让 LLM 产出**朴素暴力解 + 随机数据生成器**，**先用题面样例校验暴力解可信度**；
- 再用大量**小规模随机数据**让「用户解」与「暴力解」对拍，**第一处不一致即为最小反例**；
- 这正是竞赛选手手动「对拍」的自动化版本——即便接入 Codeforces 也拿不到它的隐藏数据，对拍恰好补上这块真值缺口。

**共性**
- 学生代码在**独立子进程**中运行，先语法编译检查区分 **CE** 与 **RE**，输出按 OJ 习惯归一化（忽略行尾空白）后比对；
- 判题状态 **AC / WA / TLE / RE / CE**，遇首个不通过即停并**给出一个具体最小反例**（输入 / 正确答案 / 你的输出），供调试 agent 接力；
- 前端用**徽章区分**本次判题走的是「🟢 真值判定」还是「🟡 差分对拍」，对拍还会标注"N 组随机数据未发现反例"（经验性结论，非数学证明）；
- 综合得分口径**极简可解释**：**得分 = 通过用例数 ÷ 总用例数 × 100**（"过了几成给几成"）；CE 记 0；无可判用例时退回静态审查分兜底。判题裁决与轨道照常展示，但不再折算成需要解释的加权分。

### 2.5 苏格拉底式分层提示

卡住时逐层求助，**4 层由浅入深**，严格按层级把控信息量、**绝不越级泄底**：

1. 仅方向性启发（不提算法名字）；
2. 点明突破口 / 数据结构方向；
3. 讲清核心算法思路与状态/转移「形状」，留关键步骤；
4. 给出较完整框架与伪代码并解释每步原因。

### 2.6 代码感知的导师对话

导师对话**像和真人助教聊天一样**，能实时看到学生的代码：

- 聊天时自动携带**编辑器里的完整代码**，导师可直接点名某一行、某个变量来讲；
- 支持**框选代码引用提问**：在编辑器里框选一段 → 右键「💬 问导师这段代码」或点对话框「⌗」按钮 →
  选中片段作为上文附进提问，导师重点针对这段回答；
- **与苏格拉底提示连贯**：对话自动带上「苏格拉底导师」已给过的分层提示作为上下文——
  学生常顺着提示里抛出的反问继续追问，若两套上下文割裂，导师就会"装作不知道自己刚问过什么"、答非所问；
  现把分层提示一并注入对话，导师顺势接着引导，不重复发问、不前后矛盾；
- 概念/技术点问答与闲聊式深入也支持（如「什么是动态规划」「Dijkstra 为什么用堆」）。

### 2.7 用户系统

- **游客优先**：不登录也能使用全部核心功能；登录后才记录历史、解锁个性化；
- 注册 / 登录：密码 **pbkdf2-HMAC-SHA256 加盐哈希**存储，登录态用 **HMAC 签名 Cookie**（无状态、防篡改）；
- 数据按用户隔离：登录用户看自己的历史与仪表盘，游客单独一个桶，互不污染。

### 2.8 用户画像 + 因材施教 🎯

- **用户画像**：从答题历史自动刻画水平**档位**（新手 / 进阶 / 高手）与强弱题型；
- **因材施教**：导师与对话的教学策略随画像自适应——
  - 新手：更多脚手架、更早更具体的提示、更多鼓励；
  - 进阶：提示精炼、多用反问、适度提高挑战；
  - 高手：简洁犀利、主动抛出更优复杂度或更难变式；
  - 针对薄弱题型主动加引导。
- > 设计原则：变的是**教学策略与挑战强度**，对每位学生始终保持尊重与鼓励，绝非「看人下菜碟」。

### 2.9 学习数据仪表盘

按用户统计并可视化：总提交 / 已攻克 / 通过率 / 平均分、**GitHub 风格刷题日历热力图**
（近一年每日提交活跃度，近 53 周 × 7 天网格，按当日提交次数分五级色阶，悬停看当天明细）、
判题结果分布（环形图）、薄弱题型、近期提交记录。
生成 AI 日报后，可**一键生成精美分享卡片图**：客户端 **Canvas** 把当日数据 + AI 寄语绘成一张
品牌化卡片，支持**保存图片 / 复制到剪贴板**，直接发到微信 / QQ / 朋友圈——平台无关、零授权、
不接微信 JS-SDK（避开「需认证主体」的合规坑）。

### 2.11 Agentic 调试回路（ReAct）

判题给出反例后，可一键召唤**调试工程师**——这不是「让模型看一眼代码说哪里错」，而是一个**会动手实验的 agent**：

- 它拥有唯一工具 `run(code, stdin)`：在沙箱里真实运行一段 Python 并拿回 stdout/stderr/状态；
- 在 **「思考 → 调用沙箱跑探针 → 观察结果 → 修正假设」** 的 ReAct 循环里多轮迭代（默认上限 4 轮）；
- 典型轨迹：先把用户代码跑在反例上确认现象 → 围绕单个假设插桩打印中间变量 → 据观察缩小范围 → 定位根因；
- 产出 **根因 + 支撑证据（哪次实验观察到的）+ 修复方向**（给思路与关键片段，但不直接甩出整题 AC 代码，保留教学性）；
- 全过程 **SSE 流式**逐步呈现到前端（💭 思考 / 🔧 探针 / 👁 观察 / ✅ 结论），**让人看见它确实在多轮调用工具**。

> 这是"真 agent"与"prompt 流水线"的分水岭：它有工具、有反馈闭环、能自我纠错，而非一次性输出。

### 2.12 代码审阅 · AI 主动反馈（行内批注 + 整段修订）

不止"你问我答"，导师能**主动改你的代码**。点「🔎 请导师审阅并给修改建议」，审阅导师通读题目与代码后：

- **行内批注**：在出问题或思路不对的**具体行**画批注（高 / 中 / 低三档严重度、对应不同颜色直接标在编辑器行上），点批注可跳转到对应行；
- **整段修订**：必要时给出一份完整可运行的修订代码，弹窗里以 **diff 对比**原代码与修订版，由你决定是否采纳；
- **应用可回退**：采纳后原代码留有快照，`Ctrl+Z` 或点「↶ 撤销，恢复我原来的代码」一键还原，绝不"覆盖了就回不去"。

定位是"老师在你卷子上批改"——既给方向，也尊重学生的选择权。

### 2.13 算法图解 · 动态可视化 🎬

面向新手友好的独立板块（顶栏「算法图解」），把抽象算法变成**逐帧动画**，看清每一步在做什么（灵感来自 Hello 算法）：

- **三套渲染器，覆盖 9 个经典算法**：
  - 数组（条形）：冒泡 / 快排 / 归并、二分查找 / 双指针；
  - 图（SVG 节点 + 边）：BFS / DFS / Dijkstra（节点状态着色、当前边高亮、Dijkstra 显示 dist）；
  - DP（填表）：0/1 背包（当前格与来源格高亮）。
- **帧式确定性播放**：每个算法把输入「编译」成一串帧（状态 + 当前伪代码行 + 旁白）。支持
  **播放 / 暂停 / 单步前进后退 / 拖动进度条 / 0.5×–4× 调速**；右侧**伪代码同步高亮当前执行行**，底部一句话旁白；
- **数组类可改输入**：手填数据或🎲一键随机，立刻重放（如二分/双指针自动排序并设目标值）；
- **与题目剖析联动**：在「辅导工作台」分析一道题后，系统按**题型 / 知识点 / 策略**文字检索到对应经典算法，
  会在剖析区挂出「🎬 相关算法图解」超链接——点一下直接切到本板块并定位到该算法看动图，把"学这道题"和"补这个算法"打通；
- **工程取舍**：纯前端 Canvas/SVG 手绘，**无图表库依赖**；刻意用**预制校验过的帧**而非 LLM 现场生成——
  可视化讲错比不讲更糟，确定性正确优先（详见 DESIGN.md §6.1）。

### 2.14 社群讨论区 🗣️

独立板块（顶栏「社群」），难题一起啃、思路互相点，支持用户上传问题与反馈：

- **发帖 / 回帖 / 点赞**：按「求助 / 题解 / 讨论 / 反馈」四板块分类筛选；帖子卡片显示作者、时间、赞数、回复数；
  详情走**右侧滑入抽屉**，可回帖、点赞（点赞**去重**：同用户同帖只计一次，靠 `UNIQUE(post_id,user_id)` 约束）；
  发帖时**标题/正文占位提示随所选板块变化**（求助引导贴报错、题解引导讲复杂度…），引导写出更对路的内容；
- **AI 审核护栏（双层）**：发帖与回帖前先审核——
  - **第一层 · 确定性关键词 + 长度过滤**：始终生效、不依赖网络，兜底拦截辱骂 / 广告引流 / 垃圾；
  - **第二层 · LLM 语义审核 agent**：捕捉关键词扫不到的语义违规，**best-effort，不可用时放行**（第一层已兜底，可用性优先）；
  - 把 UGC 的内容安全风险**变成一个可展示的工程点**，而非回避（详见 DESIGN.md §6.2）；
- **后端**：新增 `posts / post_replies / post_likes` 三张表（Alembic 迁移），登录才能发帖/回帖/点赞，首启自动灌入引导帖；
  发帖/回帖经统一的限流配额守门（审核含 LLM 调用）。

## 三、技术架构

```
                       浏览器（原生 JS · Monaco · Chart.js · SSE）
                                  │  REST / SSE
                                  ▼
┌───────────────────────  FastAPI 后端  ───────────────────────┐
│  认证(pbkdf2+HMAC Cookie) · 用户画像/因材施教 · 防滥用守门    │
│  禁缓存中间件   ·   游客/用户作用域                            │
│                                                              │
│  分析流(LangGraph)      评测流(LangGraph)     调试回路(ReAct) │
│  检索→分析→[分支]→规划   审查→[分支]→双轨判题→汇总  思考⇄跑沙箱 │
│      │                      │                      │         │
│      ▼                      ▼                      ▼         │
│  LLM 智能体(LangChain→DeepSeek)   判题引擎 judge   调试 agent │
│      │                  ┌────┴────┐                          │
│      │             真值判定     差分对拍(暴力解+生成器)        │
│      │                  └────┬────┘                          │
│  RAG 题库(TFIDF)     代码沙箱(subprocess) │ 数据(SQLite/PG)  │
│  +真题×100           +真实测试数据 data/tests/               │
└───────────────────────────────────────────────────────────────┘
```

## 四、健壮性与逻辑设计

系统对「非常规输入」与「状态一致性」做了专门处理，避免出现「输入什么都硬分析出一堆废话」的尴尬：

- **非题目守门**：分析师先判定输入是否为有效算法题（`is_problem`）。
  若是闲聊、问概念、无意义文本，则不再硬分析，给友好提示并提供「💬 让导师解答」一键转交到导师对话；
  此时「请求下一层提示」「提交评测」也统一拦截并引导，**不会硬挤无意义的提示**。
- **判定生命周期**：给「是不是有效题目」的判定加了有效期——只在「刚分析完且题面未改」时生效；
  换题 / 改题后旧判定自动作废，避免拿 A 题的判定误伤 B 题；
  提交记录也不会沿用过期的题型标签，保证画像统计准确。
- **评测流条件分支**：代码语法都过不了时跳过测试，直接收尾，节省无谓的用例生成。
- **空输入防护**：空题目、空代码等均有相应提示。

## 五、安全设计

- **密钥不进仓库**：本地放 `.deepseek_key`（被 `.gitignore` 忽略），云端用平台环境变量注入；
- **沙箱隔离**：学生代码子进程经**白名单过滤环境变量**，剥离 `DEEPSEEK_API_KEY` 等密钥，
  防止访客通过 `os.environ` 窃取密钥；
- **密码安全**：pbkdf2-HMAC-SHA256 加盐哈希，不存明文；登录态 HMAC 签名、防篡改；
- **静态资源禁缓存**：中间件给前端资源加 `Cache-Control: no-cache`，避免改版后浏览器用旧缓存导致错乱。

> 说明：沙箱为教学用途的轻量隔离（子进程 + 超时 + 临时目录 + 环境过滤），未做系统调用级硬隔离；
> 生产环境应换容器 / seccomp。

## 五·补、企业级工程化

为从「demo」向「可上线产品」演进，做了四块工程纵深：

### 1. 数据持久层（成熟化）
- **SQLAlchemy 2.0 ORM**（模型/类型/外键）替代裸 SQL，**连接池**复用连接；
- **Alembic 迁移**做 schema 版本化（已含 initial / audit_log / credits 三个迁移）；
- 同套代码本地 SQLite、线上 **Neon PostgreSQL**（`DATABASE_URL` 透明切换），启动自动 `upgrade head`。

### 2. 防滥用 · 层层核验与监察
- **IP 限流**（内存滑动窗口，20 次/分）防突发刷接口；
- **每日配额**（游客 30 / 普通用户 100 次），给 LLM 花费封顶；
- **审计日志**（audit_log 表）记录每次 LLM 调用，供监察与日报；
- **输入长度上限** + **对话话题守门**（仅答编程/算法，拒绝白嫖为通用大模型）。

### 3. 会员分级 + 模拟充值
- `credits` 算力点，`credits>0` 即 **Pro**；模拟充值（不接真实支付，零合规风险）；
- 能力差异：Pro **不受每日配额**（按算力点计）、可用**深度分析**（额外解题推演）、可生成**每日报告**。

### 4. 每日学习报告（Pro）
- 聚合当日刷题数据（尝试/通过/题型/错误/AI 互动）+ **LLM 生成叙述性点评** + 明日建议；
- 一键导出 **PDF**（reportlab 内置中文字体），全方位分析、可下载留存。

> 关于"充值"：真实支付涉及商户资质与资金合规，学生项目以**模拟积分**演示同一套"付费解锁能力"的工程架构，不接真钱。

## 六、部署

- **本地**：配置 `.deepseek_key` 或环境变量后，双击 `start.bat` 或 `python run.py`，访问 <http://127.0.0.1:8000>；
- **云端（Render）**：连接 GitHub 仓库，自动识别 `Dockerfile`，Free 套餐，配置环境变量
  `DEEPSEEK_API_KEY`（必填）、`SECRET_KEY`（建议）、`DATABASE_URL`（可选，账号持久化）；
- 端口自适应：Render 注入 `$PORT`，Hugging Face Spaces 用 `7860`，单一镜像两端通吃。

## 补充材料2 迭代（体验修复 + 商业闭环）

**体验修复（一批前端/交互断点）**
- 换题时「判题结果 / 导师审阅 / 导师对话」一并清空，杜绝残留上一题；
- 导师审阅折叠时，导师对话输入栏不再上移——对话面板吸附右栏底部，输入框稳定贴底；
- 题面支持富文本：小节标题加粗、样例代码块、幂次上标（确定性预处理 + marked）；
- 评测后题目/题型不再显示「未命名」：题库/自建题选入即记元信息，未分析直接提交也能正确命名；
- 算力点扣除后顶栏**即时刷新**，无需手动刷新页面；
- 仪表盘「近期提交」新增**时间**列；
- 社群讨论区新增**搜索栏**（标题/正文/作者）。

**功能权限分级**
- 免费（游客 + 普通登录）：导师审阅、普通智能体分析；
- Pro 算力点：深度分析、苏格拉底导师、导师对话。前后端双层拦截，非 Pro 弹解锁引导。

**模拟激励广告（商业闭环）**
- 「看广告得算力点」激励视频（倒计时赞助卡，克制编辑感设计，不接真实广告联盟）；

**题解关联**
- 发帖可选关联一道题；题目页聚合「这道题的社群讨论」，帖子详情可一键「去做这题」，做题↔复盘成回路。

**品牌中性化 / 安全收敛**
- 顶栏与分享卡去掉具体模型品牌字样（「MODEL ONLINE」）；
